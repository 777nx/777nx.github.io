1. 线路信息🚁

  • 本网站采用多线路并行部署，一份源码同时分发给多个托管商
  • 主域名：www.777nx.cn
    解析至私服，节点位于中国广州，国内速度较快(Ping值：24-27ms)
    为什么是cn的顶级域名？因为cn域名更加被人熟知，SEO等各方面都有着一定的优势
  • 备用域名1：vercel.777nx.cn
    解析至Vercel，Vercel有全球的泛播节点，兼具全球CDN功能，国内访问速度较快(Ping值：70-80ms)
  • 备用域名2：netlify.777nx.cn
    解析至Netlify，一个现代网站自动化系统，国内访问到的节点在新加坡、加州等，国内速度尚可(Ping值：90-120ms)
  • 备用域名3：cloudflare.777nx.cn
    解析至cloudflare，Cloudflare也具有全球泛播节点，其为一家位于美国的跨国科技企业，总部位于旧金山，国内速度一般(Ping值：150-200ms)
  • 备用域名4：railway.777nx.cn
    解析至Railway，一个免费容器托管平台，每个月提供5美元的免费额度，节点位于美国，国内速度一般(Ping值：200-250ms)
  • 备用域名5：render.777nx.cn
    解析至Render，一个免费容器托管平台，节点支持全球泛播，但是国内速度一般(Ping值：200-250ms)
  • 备用域名6：zeabur.777nx.cn
    解析至Zeabur，一个可以帮助你部署服务的平台，节点是位于台湾彰化的谷歌云，国内速度较快(Ping值：50-100ms)
  • 初始域名：777nx.github.io
    直链至Github泛播，国内速度较慢(Ping值：80-120ms)
  • 目前主域www.777nx.cn已劫持分流至腾讯云，在国内速度非常快(Ping值：21-35ms)
  • 以上所有线路均支持HTTPS协议

2. 技术栈-框架🧪

  • 博客框架：Hexo 6.3.0
  • 主题：Bufferfly 4.3.1
  • 源码仓库：GitHub
  • 托管与部署：本网站采用多线部署，包括Vercel + Netlify + Cloudflare + Railway + Render + Zeabur + Github Pages + 私服等
  • 评论系统：Twikoo + Vercel + MongoDB提供存储服务
  • 图床：七牛云 + 自建LightPicture
  • CDN：TianliCDN + Staticfile CDN + 75CDN + 字节跳动静态资源公共库 + elemecdn等
  • 图片压缩：Imagine + imagesTool转化图片格式为webp
  • 字体压缩：基于Python的fonttools子集化压缩 + CloudConvert压缩为woff2格式
  • 静态资源压缩：gulp + JS & CSS Minifier (Minify)
  • 自动部署：Git钩子 + Github Action
  • SEO：谷歌 + 必应 + 百度
  • 音乐平台：hexo-tag-aplayer + MetingJS
  • 流量监控：51la + 灵雀监控
  • 友链朋友圈：hexo-circle-of-friends爬虫 + SQLite提供存储服务 + Vercel部署
  • 微博热搜：weibo-top-api爬虫 + Vercel部署
  • 博客统计：不蒜子 + 百度统计
  • 异步加载与缓存加速技术：Pjax + Service Worker
  • 弹窗框架支持：Element + Vue
  • 开发语言：HTML5 + CSS3 + JavaScript + Pug + YML等
  • 版权声明：详见CC-BY-NC-SA 4.0
  • 维护日志：见旧时光栏目
  • 特别鸣谢：本站站长再此特别感谢来自阿里云的技术支持

4. 个人碎碎念🤗

  • 搭建博客初衷：作为一个科班的学生，一直觉得拥有自己的独立网站是一件十分酷的事情！这样就相当于拥有自己独立的空间，不用去CSDN这些网站上写博客或者用有道云笔记去记笔记。网站成为了自己的一个名片，在上面写写东西，记录生活，分享知识，把自己想与他人展现的技能放在博客上，何尝不是一件有趣的事情！😆😆😆

  • 博客内容：本人做的笔记与总结，主要是关于Hexo博客搭建和魔改、Java后端、Python，当然也有部分东西是搬过来的，后期可能会有一些算法的知识👏👏👏

  • 想法：为什么当初会阴差阳错地选择了Hexo这样的静态博客作为自己的网站框架呢？因为相比于Wordpress这些动态博客，Hexo可以不用购买服务器，网站的运营费用主要是域名费用(白嫖😜)。而采用动态博客就必须自己租云服务器，虽然Wordpress部署起来更加简单，不用折腾那么多(但是我是一个比较喜欢折腾的人🤣)。还有一个就是Hexo这个框架非常轻量化，启动速度很快🚀，且可拓展性十分强大，可以自己定义任何东西，非常考验自己的动手能力(虽然很多是CV的😐)。平时在电脑上装好对应的NodeJs环境和依赖，写一写Markdown文件然后直接推送到github仓库就会全自动部署，网站就可以将笔记渲染成十分好看的网页和布局，这种自动化部署和精美的页面是一种能让人十分愉快的体验(当然前期得折腾很多东西啦😏)。比如我12-2月这段时间花了大部分时间来倒腾博客(课题当然摸鱼啦😴)，逛别人博客时看到好看的就想搬到自己的网站上😂，最后基本上就弄成现在这个样子。在魔改博客的过程中也认识了非常多志同道合的小伙伴，也少不了各路前端大佬的帮助和指导，在此表示非常感激！在这几个月以来，自己对于Butterfly主题的了解也在逐渐加深，认识到主题作者这种大神的前端功底是如此之深！我自己也从一只不敢动源码的小菜鸡到现在会自己改一些简单源码，在原来主题的基础上进行二次开发。这种能力的提升要全身心去投入才会感受到，这也意味着博客的样式可以根据自己的想法去实现，打造一个完全属于自己的网站。
